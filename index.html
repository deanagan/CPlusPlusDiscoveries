Why use the STL?

  • It is well tested and optimised.
  • Don't reinvent the wheel.
  • It leads to shorter more compact code.

STL Approach

  • There is preference for free functions over member functions (e.g. std::find(container..) over container.find() ).
  • Algorithms are designed to work for most containers.
  • Algorithms work with iterators.

Example Algorithm: std::find_if

Finding the first odd number in a sequence of numbers
                        

                            std::vector<int> n { 2, 4, 6, 9, 10 };

                            auto result_iter = std::find_if(begin(n), end(n), ???);

Before C++11, we had 2 common approaches

Using a stand alone/free function
                        
bool IsOdd(int n) {
  return n % 2 != 0;
}
                        
                        
                        
void printOdd() {
  std::vector<int>::iterator it; // no auto yet!
  it = std::find_if(n.begin(), n.end(), IsOdd);

  if (it != n.end()) {
    std::cout << *it;
  }
}

Or using a function object or functor

                        
struct IsOdd {
  bool operator()(int n) const {
    return n%2 != 0;
  }
};
                        
                        
                        
void printOdd() {
  std::vector<int>::iterator it; // no auto yet!
  it = std::find_if(n.begin(), n.end(), IsOdd());

  if (it != n.end()) {
    std::cout << *it;
  }
}

With Modern C++(C++11 and beyond), we now have lambdas.

                        
void printOdd() {  
  auto it = std::find_if(begin(n), end(n), [] (int n) { 
      return n%2 != 0;
  });

  if (it != n.end()) {
    std::cout << *it;
  }
}
